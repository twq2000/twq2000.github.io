    • Windows

      1. 高级系统设置 -> 环境变量 -> 系统变量，新建一个名为 JAVA_HOME 的变量名，其值为 jdk 的安装目录路径。

        这里是一个示例：

      2. 在系统变量的 Path 变量中，添加一条记录 %JAVA_HOME%\bin。

    • Mac

      1. 在终端窗口中，创建并打开全局环境变量文件

      2. 在 .bash_profile 文件中进行编辑，配置 jdk 环境变量

  2. 检验配置是否成功

    现在，让我们在终端输入 java -version 来测试是否一切都正常。如果能正确显示版本号，那么就配置成功了~ 否则，就需要重新检查 JAVA_HOME 和 Path 中的路径配置是否有问题。


其实还有一个叫做 classpath 的配置项，不过官方表示：jdk1.5以后就不用配置它了。

Redis

安装阶段

  • Windows

    Redis官网建议使用Linux进行部署，未提供Windows版本的Redis。

    但微软开发和维护着Windows64版本的Redis，可以在GitHub上进行下载：microsoftarchive/redis

    1. 下载 .zip 的压缩包文件，并解压至任意目录
    2. 启动服务器程序 redis-server.exe，并保持窗口开启
    3. 启动客户端程序 redis-cli.exe，然后执行 ping 命令。如果显示 PONG，那么说明连接成功。

Maven

安装阶段

官网安装地址：Maven – Download Apache Maven

安装阶段没有太多需要额外说明的。


配置阶段

maven的使用是在jdk的基础上，因此，请先确保自己电脑已经有了jdk环境。

  1. 配置环境变量

    • Windows

      1. 高级系统设置 -> 环境变量 -> 系统变量，新建一个名为 MAVEN_HOME 的变量名，其值为 maven 的安装目录路径。

        这里是一个示例：

      2. 在系统变量的 Path 变量中，添加一条记录 %MAVEN_HOME%\bin。

    • Mac

      1. 在终端窗口中，创建并打开全局环境变量文件

      2. 在 .bash_profile 文件中进行编辑，配置 jdk 环境变量

  2. 检验配置是否成功

    打开终端，输入命令 mvn -v 如果能正确显示版本号，那么就配置成功了~

  3. 配置本地仓库的存储路径

    1. 在maven的安装目录下，创建一个文件夹，用于作为仓库。（一般将这个文件夹起名为 repository）

    2. 在核心配置文件 conf/settings.xml 中，将本地仓库路径设置为对应路径即可。

      这里是一个示例：<localRepository>E:\develop\apache-maven-3.8.3\repository</localRepository>

  4. 配置中央仓库地址

    在mirrors节点添加新的mirror子节点，以下内容截取自我自己的配置文件：

    如果你想查看之前配置过的东西，可以使用 git config --list 命令来列出所有 Git 可以找到的配置项。

    如果你是有针对性的查找某一项配置，也可以直接使用 git config <key> ，如 git config user.name 。
